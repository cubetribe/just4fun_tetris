# Contributing

Danke für dein Interesse am Projekt.

## Workflow

1. Fork erstellen oder Feature-Branch öffnen
2. Kleine, fokussierte Änderungen umsetzen
3. Manuell auf Mobile + Desktop testen
4. Pull Request mit klarer Beschreibung einreichen

## PR-Checkliste

- UI auf Mobile getestet
- Gameplay nicht regressiv (Drop, Rotation, Kollision)
- Keine neuen globalen Side Effects
- README bei Feature-Änderungen aktualisiert

## Commit-Konvention (empfohlen)

- `feat:` neue Features
- `fix:` Bugfixes
- `refactor:` interne Verbesserungen
- `docs:` Dokumentation

## Code-Style

- Vanilla JS, klare Funktionen, keine unnötigen Abhängigkeiten
- CSS mobile-first
- Lesbarkeit vor Cleverness
